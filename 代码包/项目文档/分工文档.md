## 二、后端（同学1）— 详细任务

### 2.1 基础架构搭建

### 2.4 后端安全要点（重点关注）

| 安全项 | 涉及接口 | 要求 |
|--------|----------|------|
| 身份伪造防护 | API203, API205 | `userId` 从 `sessionToken` 解析，**禁止**客户端传入 |
| openid 不暴露 | API101, API102 | `openid` 仅存 Redis，使用 `bindToken` 中转 |
| SSRF 防护 | API105 | 校验 URL 白名单 + 拒绝内网地址 |
| 跨商铺核销 | API410 | 校验 `merchantId` 一致性 |
| 超卖防护 | API115 | 悲观锁 `SELECT FOR UPDATE` 或乐观锁版本号 |
| 登录暴力破解 | API301, API402 | 图形验证码 + 5次/5分钟锁定 |
| 短信轰炸 | API0001 | 60秒/次 + 10次/天 + 图形验证码前置 |
| 文件上传 | API0002 | MIME 白名单 + 10MB 上限 + ClamAV 扫描 |

### 5.3 关键数据流

```
用户投放垃圾
  → 终端抓拍上传
  → API203 (后端) → YOLO 推理 (AI) → 分类结果
  → 积分结算 → 写入 DeliveryRecord + PointRecord
  → API204 设备状态上报
  → 小程序推送积分通知

用户拍照查询
  → 小程序拍照 → API0002 上传 → API105 (后端) → YOLO 推理 (AI)
  → 返回检测框 + 分类 + 投放指引 → 小程序渲染

月度报告生成
  → API333 (后端) → 数据聚合 → LLM 分析 (AI)
  → Markdown 报告 → 结构化校验 → 存入 AnalysisReport
  → 管理员/商家查看
```

## 六、协作流程与规范

### 6.1 开发顺序建议

```
第 1 阶段 ─── 基础搭建 ───
  同学1: Flask 框架 + 数据库建表 + 鉴权中间件
  同学2: 前端项目脚手架 + 请求封装 + 布局框架

第 2 阶段 ─── 核心链路 ───
```

### 6.2 接口联调规范

- 后端接口开发完成后，在接口文档中标记为 **"已完成"**
- 前端优先使用 Mock 数据开发页面，后端接口就绪后切换真实接口
- AI 推理服务独立部署，先提供 HTTP 接口文档给后端进行联调
- 联调问题在接口文档中直接注释，不做口头传递

### 6.3 代码管理规范

- 分支策略：`main` 保护分支，每人开发在 `feature/<模块名>` 分支进行
- 提交规范：`[模块] 简要说明`，如 `[user] 完成微信登录接口`
- 接口文档同步：接口变更必须同步更新 `项目文档/API接口文档.md`
- 每日站会：同步进度、阻塞点、接口对接状态
